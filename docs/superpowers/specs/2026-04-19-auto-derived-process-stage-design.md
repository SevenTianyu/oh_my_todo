# 自动推导流程阶段设计

## 概述

当前项目把 `stage` 作为流程的手填字段，同时又允许用户通过轮次时间、轮次状态和手动归档去表达真实进展。这导致“流程阶段”成为一套额外维护的状态，和实际使用心智不一致。

本次改动将把流程阶段从“用户维护的持久化字段”改成“由流程轮次自动推导出的运行时视图”：

- 活跃流程只存在两种推导阶段：`筛选中`、`面试中`
- 归档继续由用户手动触发；一旦归档，该流程不再参与阶段概念，也不再需要 `closed`
- 导入旧 JSON 时继续接受历史 `stage` 字段，但运行时忽略它；后续导出时不再写出该字段

目标是让产品行为与用户心智一致，同时在不打断现有本地数据使用的前提下，悄悄完成 JSON 格式升级。

## 目标行为

### 1. 阶段不再手填

- 新建公司表单移除“流程阶段”字段
- 公司卡片不再显示流程阶段 badge
- 流程数据模型不再要求创建或更新 `stage`

### 2. 阶段由轮次自动推导

对每个活跃流程，运行时按以下规则推导阶段：

- `面试中`
  - 只要该流程中任一轮体现“发生过面试推进”，就判定为 `面试中`
- `筛选中`
  - 活跃流程中所有轮次都还没有体现“发生过面试推进”

“发生过面试推进”的判定规则为满足以下任一条件：

- `round.scheduledAt !== null`
- `round.status` 为 `scheduled`
- `round.status` 为 `completed`
- `round.status` 为 `waiting-result`
- `round.status` 为 `closed`

这个规则保证以下行为成立：

- 只要用户给任一轮填过时间，就会进入 `面试中`
- 即使后来清掉时间，只要轮次已经处于推进后的状态，流程仍保持 `面试中`
- 只在完全没有推进痕迹时，流程才算 `筛选中`

### 3. 归档就是结束

- 用户继续通过现有“归档流程”操作结束一个流程
- 被归档的流程不再出现在活跃区，也不参与阶段分组
- 系统不再维护 `已结束` 或 `closed` 这类额外流程阶段
- “归档”本身就代表用户已放弃，不区分失败、拒绝 offer 或主动停止

## 界面影响

### 1. 新建公司表单

- 移除“流程阶段”选择框
- 新建公司时只保留最小必填字段：
  - 公司名称
  - 公司类型
  - 岗位名称
- 首轮默认名继续沿用当前最小录入策略，默认创建第一轮待推进轮次
- 文案不再暗示用户需要手动维护流程阶段

### 2. 活跃流程工作区

- 保留“按流程阶段分组”入口
- 该分组只保留两组：
  - `筛选中`
  - `面试中`
- “按公司类型分组”保持不变
- 公司卡片中的流程 badge 不再展示阶段标签

### 3. 归档区

- 归档区结构保持不变
- 归档后的流程仅通过“归档区”体现，不再显示“已结束”流程阶段

## 数据与兼容

### 1. 主数据模型

- `InterviewProcess.stage` 从运行时主模型中移除
- 相关推导逻辑通过 selector 或专用 helper 在运行时计算
- `ProcessStatus` 继续保留 `active | archived`，用于区分是否在活跃区

### 2. 导入兼容

- 解析旧本地 JSON 时，允许输入中继续包含 `stage`
- 该字段不会阻塞导入，也不会参与运行时逻辑
- 旧数据导入后，阶段展示和分组将基于轮次自动推导，而不是旧值

### 3. 导出升级

- 未来导出的 JSON 快照不再包含 `stage`
- 这会让用户在无感知的情况下完成本地数据格式升级
- 不做额外迁移提示，不要求用户手动重建数据

## 实现边界

### 保留

- `nextStep` 字段仍保留
- 轮次状态仍保留现有枚举
- 归档流程的交互方式保持不变
- “按流程阶段分组”入口保留

### 移除或收敛

- 移除 `offer` 和 `closed` 作为流程阶段的产品概念
- 移除新建表单中的阶段选择
- 移除公司卡片里的阶段 badge
- 移除导出 JSON 中的 `stage`

## 测试与验收

### 自动化测试

- 选择器测试：
  - 无推进痕迹的活跃流程被推导为 `筛选中`
  - 有 `scheduledAt` 的流程被推导为 `面试中`
  - 有 `completed / waiting-result / closed` 状态的流程即使无时间也被推导为 `面试中`
  - 已归档流程不会进入活跃分组
- 表单/应用测试：
  - 新建公司不再显示“流程阶段”
  - 新建公司后在未安排任何时间前处于 `筛选中`
  - 给轮次设置时间后，切到“按流程阶段分组”能看到它出现在 `面试中`
- 存储测试：
  - 导入带 `stage` 的旧 JSON 仍成功
  - 导出后的新 JSON 不再包含 `stage`

### 手工验收

- 新建公司时不需要再思考流程阶段
- 只要给某一轮安排了时间，流程就自然进入 `面试中`
- 手动归档后，该流程从活跃区消失并进入归档区
- 导出一个旧数据升级后的快照，JSON 中不再出现 `stage`

## 非目标

- 不新增自动归档能力
- 不根据“面试失败 / 谈薪拒绝 / 主动放弃”做细分状态
- 不改动本地存储 key
- 不在本次引入新的迁移版本号机制，除非实现时发现当前导入/导出结构必须显式升级版本
